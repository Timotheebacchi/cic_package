introduction-cic

library(cic)

Overview

cic estimates the Changes-in-Changes parameter and returns several types of confidence intervals. The package also includes a diagnostic helper to check whether the observed data are broadly compatible with the assumptions used by the estimator.

In this vignette we use the built-in simulation helpers to show the full workflow:

  1. generate data,
  2. run the diagnostic check,
  3. estimate the CiC parameter with several interval methods,
  4. inspect the output,
  5. visualize the confidence intervals.

The simulation helpers are mainly intended for examples and Monte Carlo experiments. They let you quickly reproduce the DGP used in the package documentation and tests.

Diagnostic Check

Before estimating the parameter, it is useful to inspect the input data with check_cic_assumptions().

diag <- check_cic_assumptions(d$Y, d$X, d$Z)
diag$pass_all
#> [1] FALSE
diag$messages
#> [1] "Convergence Alert [Assumption 2(iv)]: The combined tail heaviness of your data (b + d ≥ 0.5) exceeds the theoretical threshold. The estimator will experience severe convergence slowdowns or non-normal asymptotics. (Left: b₁+d₁=0.569, Right: b₂+d₂=0.043)"

The diagnostic output is a quick screening tool, not a formal proof that the model is correct. It is most useful as a pre-check before interpreting the point estimate and confidence intervals.

Estimation

The method argument accepts one or several interval constructors.

fit <- cic(
  d$Y, d$X, d$Z,
  method = c("no-split", "split", "kde", "bse", "bpc"),
  B = 500
)

The first three methods are asymptotic constructions:

  • no-split: full-sample plug-in variance estimator
  • split: sample-splitting variance estimator
  • kde: kernel-density-based variance estimator

The bootstrap methods are included for robustness checks and finite-sample comparisons:

  • bse: bootstrap standard-error interval
  • bpc: bootstrap percentile interval

Visualization

The package provides a plot method to compare the confidence intervals visually.

plot(fit)

Practical Notes

For empirical work, the main workflow is:

  1. inspect the sample with check_cic_assumptions(),
  2. estimate with at least one asymptotic method,
  3. compare with a bootstrap method if you want a robustness check,
  4. report the chosen method and confidence level explicitly.

The vignette and README now follow the manuscript by Chhor, D’Haultfoeuille, L’Hour, and Mugnier rather than the older Athey-Imbens reference.
